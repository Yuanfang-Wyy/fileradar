# Scout Roadmap

Scout 的演进主线：从 **文件名检索**（Everything 路线）走向 **内容 + 语义 + 智能检索**（Spotlight / 现代 AI 搜索路线）。

## 核心设计：搜索模式可配置

搜索能力分档，重的能力按需开启，用户在设置里选择当前模式：

| 模式 | 说明 | 成本 |
|---|---|---|
| **文件名**（默认） | 文件名/路径匹配，秒级索引、毫秒搜索 | 轻 |
| **文件内容** | 提取 PDF/Office/文本/代码正文做全文索引 | 中（索引体积、提取耗时） |
| **语义** | embedding 向量检索，按语义相关度召回 | 重（模型、向量库） |
| **自然语言** | LLM 解析查询 + RAG 问答 | 重（依赖 LLM API） |

架构上 `searcher` 按模式分发，各模式实现独立、可渐进上线；UI 提供模式切换。

---

## 第一梯队 · 搜索能力的质变

- [ ] **中文拼音首字母匹配**：`yddz` → 「移动底座」。索引期算拼音存列，搜索期一并匹配。← 进行中
- [ ] **中文分词**：FTS5 默认按单字切，接 jieba/nodejieba 或 trigram tokenizer 提升中文召回
- [ ] **模糊匹配（fuzzy）**：`fbar` → `Scout`（子序列），SQLite 召回后 fzf 式打分重排
- [ ] **文件内容全文索引**：pdf/docx/xlsx/md/code 提取正文 → FTS5；增量、可排除大文件

## 第二梯队 · AI 时代

- [ ] **语义搜索**：内容分块 → embedding → 向量检索（sqlite-vec / 本地或云端模型）
- [ ] **自然语言查询**：LLM 把「上周改过的大于 1MB 的 PDF」解析为结构化查询
- [ ] **RAG 问答**：「在我的文件里找答案」，命中片段 + 引用来源

## 第三梯队 · 启动器化体验

- [ ] **快速操作（Actions）**：搜到即操作——重命名 / 移动 / 压缩 / 分享 / 指定 app 打开
- [ ] **Quick Look 预览**：空格预览选中文件
- [ ] **frecency 排序**：按使用频率 + 最近度排序，而非纯修改时间

---

## 工程 · 可扩展性与分发

- [ ] **watcher 换 FSEvents**：chokidar 逐文件监听百万级目录不可持续（内存/句柄），改用 macOS FSEvents 系统级变更流；编译 fsevents
- [ ] **索引提速**：串行 `stat` → 并发限流；全量重扫 → 增量对比 mtime
- [ ] **搜索 COUNT 优化**：大结果集每次全量 `COUNT(*)` 慢，改为上限计数 / 异步
- [ ] **分发**：Apple 代码签名 + 公证（免右键打开）、GitHub Actions CI（自动 test/build/release）、electron-updater 自动更新、universal binary 兼容 Intel

---

> 优先级取舍：先做对中文场景立竿见影的 **拼音匹配**，再上 **内容全文** 完成定位升级，让 Scout 超越「Everything 的 Mac 复刻」。
